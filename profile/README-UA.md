# Concept‑Labs

## Створюйте корпоративні PHP додатки з ясністю та впевненістю

**Concept-Labs — це сучасна PHP екосистема, яка усуває складність традиційних фреймворків, зберігаючи можливості корпоративного рівня.**

> Припиніть боротися з магією та прихованими конвенціями. Створюйте додатки, де кожен елемент конфігурації прозорий, кожна залежність явна, і кожен компонент тестується.

---

## Чому існує Concept-Labs

Після десятиліть розробки PHP у корпоративних додатках, наш засновник [Віктор Галіцький](https://github.com/vgalitsky) став свідком тривожної тенденції: **сучасні PHP фреймворки перетворилися на непрозорі чорні скриньки**. Конфігурація розкидана по десяткам файлів. Залежності вирішуються через приховану магію. Поведінка визначається конвенціями, які потрібно запам'ятати або відкрити методом спроб та помилок.

**Реальна ціна?** Години налагодження таємничої поведінки. Дні відстеження ланцюжків залежностей. Тижні боротьби з обмеженнями фреймворків при масштабуванні на production.

**Concept-Labs — це відповідь:** Філософія, дистильована з важко здобутого досвіду, кристалізована в трьох принципах:

1. **🔍 Прозорість замість магії** - Кожна конфігурація розповідає історію. Кожна залежність явна. Ніяких сюрпризів, ніколи.
2. **🧩 Композиційність замість монолітів** - Невеликі, сфокусовані пакети, які роблять одну річ бездоганно. Складайте свій ідеальний стек.
3. **🚀 Production-First дизайн** - Створено для реального світу: довготривалі процеси, висока конкурентність і безкомпромісні production середовища.

Це не просто черговий фреймворк—це **фундаментальне переосмислення PHP архітектури додатків**, народжене з десятиліть бойових шрамів і відточене в елегантну, потужну екосистему.

## Що робить Concept-Labs особливим

### 📖 Конфігурація, що розповідає історію
Відкрийте будь-який конфігураційний файл і миттєво зрозумійте архітектуру вашого додатку. Немає полювання по десяткам файлів чи розшифрування магічних конвенцій. Все явне, читабельне і дружнє до системи контролю версій. Підтримка JSON, PHP, з XML та YAML незабаром.

### 🔌 Справжня Drop-In інтеграція
Додайте будь-який пакет Concept-Labs через Composer, і він автоматично інтегрується. Кожен пакет несе свій власний `concept.json` маніфест—самоописуючі залежності та конфігурацію. Нуль ручного з'єднання. Нуль сюрпризів.

### ✨ PSR стандарти скрізь
Побудовано на індустріальних стандартах (**PSR-7, PSR-11, PSR-14, PSR-15, PSR-17**), що означає: ваш код безперебійно працює з більш широкою PHP екосистемою. Напишіть раз, інтегруйте всюди. Захистіть майбутнє ваших додатків.

### 🚀 Продуктивність production-рівня
Розроблено для довготривалих процесів, таких як RoadRunner та Swoole. Низькі накладні витрати middleware, ефективне управління пам'яттю та непробивні межі помилок забезпечують масштабування вашого додатку, коли це найбільш важливо.

## Екосистема

Наша екосистема побудована на ретельно спроектованих, сфокусованих пакетах, які вирішують реальні проблеми:

### Фундаментальний рівень
- **[@Concept-Labs/config](https://github.com/Concept-Labs/config)** - Перетворіть хаос конфігурації на ясність за допомогою інтелектуальної JSON конфігурації зі змінними оточення, імпортами та перехресними посиланнями
- **[@Concept-Labs/singularity](https://github.com/Concept-Labs/singularity)** - Візьміть контроль над залежностями за допомогою PSR-11 контейнера, що усуває здогадки через явні переваги

### Комунікація та події
- **[@Concept-Labs/event-dispatcher](https://github.com/Concept-Labs/event-dispatcher)** - Розв'яжіть ваш додаток за допомогою перевіреної PSR-14 системи подій, що тримає компоненти слабко зв'язаними

### HTTP стек
- **[@Concept-Labs/http-message](https://github.com/Concept-Labs/http-message)** - Сучасні PSR-7/PSR-17 HTTP повідомлення, створені для PHP 8.2+ без компромісів
- **[@Concept-Labs/http](https://github.com/Concept-Labs/http)** - Готовий до корпоративного використання HTTP стек з пріоритетним конвеєром middleware для складних додатків
- **[@Concept-Labs/simple-http](https://github.com/Concept-Labs/simple-http)** - Запустіть HTTP додатки за хвилини з нашим шаблоном швидкого старту

### Рівень даних
- **[@Concept-Labs/dbal](https://github.com/Concept-Labs/dbal)** - Пишіть безпечні, підтримувані запити до бази даних з плавними конструкторами, що запобігають SQL ін'єкціям
- **[@Concept-Labs/orm](https://github.com/Concept-Labs/orm)** - Мапте об'єкти на бази даних без жертвування контролем або прозорістю

### Утиліти
- **[@Concept-Labs/arrays](https://github.com/Concept-Labs/arrays)** - Опануйте складні структури даних з розширеними утилітами крапкової нотації та рекурсивних масивів
- **[@Concept-Labs/cli](https://github.com/Concept-Labs/cli)** - Створюйте професійні інструменти командного рядка з елегантними API
- **[@Concept-Labs/simple-cache](https://github.com/Concept-Labs/simple-cache)** - PSR-16 кешування зробленe простим і надійним

**70+ пакетів і зростання** — Кожен корисний незалежно, всі безперебійно інтегровані.

## Запустіть свій перший додаток за 5 хвилин

```bash
composer create-project concept-labs/simple-http my-app
cd my-app && cp etc/default.json.dist etc/default.json
php -S localhost:8000 -t pub
```

**Готово.** Ніякого створення каркасу. Ніякої генерації коду. Ніяких таємниць. Просто робочий додаток, готовий для ваших ідей.

## Хто повинен обрати Concept-Labs?

### ✅ Ідеально для:
- **Досвідчених розробників**, які цінують явну поведінку над неявною
- **Зростаючих команд**, що потребують читабельного, підтримуваного коду
- **Критично важливих для production додатків**, що вимагають високої продуктивності та надійності
- **Мікросервісних архітектур**, які отримують вигоду від композиційних компонентів
- **Довготривалих PHP додатків** на RoadRunner, Swoole чи подібних
- **API-орієнтованих проєктів**, що потребують надійного HTTP middleware

### 💡 Розгляньте альтернативи, якщо:
- Ви віддаєте перевагу фреймворкам "конвенція над конфігурацією", таким як Laravel
- Вам потрібна адмін-панель з усім необхідним (розгляньте AdminLTE + ваш фреймворк)
- Складне управління відносинами ORM є вашим пріоритетом (Doctrine може підійти краще)
- Ви будуєте швидкий прототип і не потребуєте довгострокової підтримуваності

**Чесність будує довіру.** Concept-Labs бездоганно вирішує конкретні проблеми—переконайтеся, що це ваші проблеми.

## Філософія на практиці

> **"Ваш додаток повинен бути розмовою між конфігурацією та кодом—а не археологічною експедицією."**  
> — Філософія Concept-Labs

Кожне рішення в Concept-Labs випливає з реальних болючих точок, які ми пережили:

- **Налагодження додатків, де поведінка виникала з прихованих конвенцій** → Конфігурація тепер розповідає повну історію
- **Відстеження залежностей через внутрішні частини фреймворку** → Залежності явні та простежувані
- **Боротьба з обмеженнями фреймворку при масштабуванні** → Вимоги production керують кожним дизайнерським рішенням
- **Онбординг розробників, яким потрібні тижні, щоб стати продуктивними** → Нові члени команди розуміють систему за години

Concept-Labs існує, тому що **корпоративні додатки заслуговують на краще, ніж магічні трюки**.

**Будуйте з ясністю. Розгортайте з впевненістю. Масштабуйте без сюрпризів.**
